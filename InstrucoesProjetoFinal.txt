Classe Dao
● Desenvolver um sistema orientado a objetos é criar classes e
atribuir responsabilidades
● Precisamos de uma classe responsável por inserir, alterar,
consultar, excluir e listar os objetos salvos no banco
● Essa classe será a Dao (ou qualquer outro nome que você preferir)
● Dao - Data Access Object: encapsula o acesso ao banco para que a
aplicação possa se comunicar com ele

Classe Dao
● Em outras palavras:
○ O restante da aplicação não precisa saber como realizar as
operações
○ Basta instanciar um objeto Dao e chamar os métodos
apropriados…
● Não precisamos saber os detalhes da classe, apenas seus
métodos
○ O que eles recebem e o que eles retornam

Classe Dao
● Na aplicação precisamos persistir usuários, motoristas, veículos…
● Mesmas operações, objetos diferentes
● Situação para classe genérica
○ Quando quisermos persistir um objetos, utilizamos o tipo
apropriado
Dao<Veiculo> daoVeiculo = new Dao(Veiculo.class);
Dao<Usuario> daoUsuario = new Dao(Usuario.class);

Classe Veiculo
● Mongo salva os objetos no formato BSON
● BSON: Binary JavaScript Object Notation (Binary JSON)
● Formato para armazenar documentos binários no MongoDB
● Constituído de chave + valor

Observações
● A classe Dao não faz tratamento de exceções
○ Exemplo: não deve ser permitido inserir dois veículos com
a mesma placa
● Podem ser necessários outros métodos na classe Dao
● Não se preocupe com a estética (por enquanto)
